2.1 基本内置类型

首先要明确的是C++是一门静态类型语言，与之对应的动态类型语言是python之流。没转专业以前自学Python的时候不知道动态语言的解释到底是个啥东西。直到之前刚好用Python写了个稍微有点模样的爬虫，再回过头来对比C++，恍然大悟：噢！原来是这个意思！
简言之，静态类型语言的每一个变量都要有明确的类型，这样编译器才能够准确识别；而动态类型语言的变量类型是在运行时指定的。相比起来动态类型的确是稍微灵活一些。看一个例子

暂且不说C++以分号来分开每一行，python使用缩进来识别每一行； 可以发现C++在每个变量名的面前都要有一个类型，来告诉编译器：这个是string字符串，这个是int类型、这个是double类型、这个是字符数组……等等。而Python就直接赋值，编译器看到之后：噢，这个是int、这个是double、这个是一个数组（类型我还不知道，他待会放什么就是什么吧）；大概是这样子的意思。。。

那么既然是静态语言，就要用上既定的格式：声明变量时别忘了带上他的类型；
而C++提供了内置的基本类型，也允许我们定义自己要用的类型，这就是“class”-“类”；类就是OOP的精华了吧（个人粗鄙地认为），目前尚未讲到，但是我提一下：类就是数据+对数据的操作，两个东西封装在一个黑盒子里，用户可以根据说明书使用这些“操作方法”，而具体实现的过程用户是不需要考虑的。

2.2 变量

第二小节讲了变量。变量其实就是你设置的一个个名称，用来存储相应的值。在C++中，变量要先声明再引用，不能够凭空用上一个未声明的变量；这就引出了“声明”和“定义”的区别。
按照书上的说法：“声明”是告诉程序有这么一个变量存在，但是还没给变量申请实际的存储空间，也就是说他只是一个名号而已。而“定义”则是给变量申请了实实在在的空间，哪怕存的值是0，他也占用了实际的空间；通过“声明”&“定义”，我们能将程序进行逻辑分层分块（分离式编译），在最基础的模块中【定义】一系列要共享的变量或者方法，在他的子模块中就可以通过【声明】而使用了。“一个变量可以声明多次，但只能定义一次！”

2.3 复合类型

复合类型比较有意思，主要就是讲了两个有趣的玩意：引用和指针。都说指针很难理解， emmm， 一开始我学C时也这么觉得的，现在重新静下心来看一看教材，条理还是比较清晰的！
用一下练习2.19我写的答案

//exe2.19 请说明指针和引用的主要区别

引用：初始化时不能定义为空，必须与一个已经存在的，类型相同的对象相绑定。并且绑定之后不能更改。引用本身不占内存空间，并不是一个实体存在的对象。对引用做的所有修改其实就是在对其绑定的对象进行修改。

指针：本身是一个对象，也有相应的基本类型，指针和对象若类型不同，不能联系在一起。指针可以更改指向的对象。指针存放的是对象的地址。要给指针赋值（改变其指向的对象），需要使用“取地址符&”；要给指针指向的对象赋值，需要使用“解引用符*”
称该操作为：访问

引用本身不能为空，编译器会报错。 指针可以为空，（nullptr / 0 / NULL）；
建议指针在声明的同时定义（初始化），不要让指针指向未知的内存空间。

复合类型有这两个有趣的玩意，对应的操作符也很有趣，一开始的确没少让我抓狂，那就是“取地址符‘&’”和“解引用符‘*’”，在上文也有提到。在这里也先不展开说了（主要是自己没怎么经历过艰难岁月
